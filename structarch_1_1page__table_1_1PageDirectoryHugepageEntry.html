Detailed Description

4MB page directory table hugepage entry.

Note
The only hugepage usage in HmmOS is set up by the bootloader for the HHDM. For simplicity (for now), HmmOS doesn't attempt to map/unmap any hugepages, and only dealing with 4KB pages. Any attempt to modify the existing HHDM mapping/allocate new hugepages is UB. This struct only exists to properly enumerate the bootloader's HHDM. If we ever want to allow user-space hugepages, we should look into libhugetblfs.
