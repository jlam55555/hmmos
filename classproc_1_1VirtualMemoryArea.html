Detailed Description

Representation of a memory mapping, a.k.a. "virtual memory area" or VMA. Akin to Linux's vm_area_struct.

Like fs::File, this is owned by a single process and maintains a reference count on a dentry. A VirtualMemoryArea can only be created from an open fs::File, but does not require that the fs::File be kept open.
